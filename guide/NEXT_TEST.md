# 회사 PC 도착 후 즉시 테스트 (Step 0 룰 v2 검증)

작성: 2026-05-06 (사용자 부재 시간 동안)

---

## 두괄식

1. `verify.bat` 더블클릭 → byte OK 확인 (예상 64,900~65,000)
2. CX 시스템 프롬프트 사내 챗봇 **재등록** (덮어쓰기)
3. 아래 13개 case 챗봇에 그대로 입력 → 분류 + 교정 결과 확인

---

## 사전 — verify.bat 실행

byte FAIL 나오면 즉시 알려주세요. 절감 작업.

## VOC 5건 (사용자 핵심 비판 사례 + 변형)

### V1. 사용자 원본 사례
```
고객님 불편을 드려 거듭 사과드립니다. 대단히 죄송합니다. 진심으로 사과드립니다.
```
**기대**: G 매칭 → VOC 분류 → [교정 결과] 출력 (4단 자연어, 사과 1회로 압축, 거듭·대단히·진심으로 삭제)
**FAIL 신호**: "유형 확인 부탁드립니다" 출력 → 룰 v2 미작동

### V2. 콜센터 민원 답변
```
안녕하세요 김미래 고객님. 콜센터 응대 과정에서 불편을 드린 점 깊이 사과드립니다. 응대 품질을 점검하고 강화하겠습니다.
```
**기대**: G 매칭 → VOC 분류 (콜센터·불편·사과·점검·강화 다중 매칭)

### V3. 거절형 VOC
```
처리 어려움을 안내드리려 합니다. 너른 양해를 부탁드립니다. 다른 방안을 검토하겠습니다.
```
**기대**: G 매칭 → VOC 거절형 (양해·검토 매칭)

### V4. 짧은 사과 (edge)
```
사과드립니다.
```
**기대**: G 매칭 → VOC (사과드립니다 1개 + 말머리 0 + ■ 0)

### V5. 사용자 명시
```
VOC 답변 초안:
김미래 고객님, 안녕하세요. 처리해 드리겠습니다.
```
**기대**: 1단계 사용자 명시 → VOC (즉시 채택, 2단계 안 봄)

## LMS 고지/공지 4건

### L1. 정답데이터 LN102 ASIS
```
[미래에셋대우] 융자만기일 안내
#{고객명} 고객님
#{계좌번호}
#{종목명}의 융자 만기일을 알려드립니다.
☎고객센터: 1588-6800
```
**기대**: F 매칭 → LMS 고지 → 정답 TOBE처럼 [미래에셋증권]·#{고객명} 고객님,·■ 계좌정보·■ 융자만기정보·■ 문의 변환

### L2. 짧은 LMS
```
[미래에셋증권] 잔고 조회 안내
#{고객명} 고객님, 잔고를 확인해 주세요.
```
**기대**: F 매칭 → LMS 고지 (또는 본문 짧으면 E SMS — 둘 다 OK)

### L3. ASIS 한자어·옛 어투
```
[미래에셋증권] 만기 안내
#{고객명} 고객님
당사는 익영업일에 만기처리 진행하오니 양지하시기 바랍니다.
☎고객센터: 1588-6800
```
**기대**: F 매칭 → LMS 고지 → "당사→미래에셋증권"·"익영업일→다음 영업일"·"양지하시기 바랍니다→참고해 주세요"·☎→■ 문의

### L4. 다양 ■ 블록
```
[미래에셋증권] 추가담보 납부 안내
#{고객명} 고객님
담보부족 #{대상금액}원이 발생하였습니다.
■ 대상계좌: #{계좌번호}
■ 부족금액: #{대상금액}원
☎고객센터: 1588-6800
```
**기대**: F 매칭 → LMS 고지 (■ 블록 + 계좌·담보 enum 정확히 적용)

## LMS 마케팅 2건

### M1. (광고) 명시
```
(광고)[미래에셋증권] #{고객명} 고객님, 신규 계좌 개설하면 5만원 혜택을 드려요!
지금 바로 신청하세요.
※ 투자 전 설명 청취
```
**기대**: A 매칭 → LMS 마케팅 → 심의문구 3종 verbatim·해요체 유지

### M2. 마케팅 키워드 다수
```
[미래에셋증권] 신규 가입 이벤트 안내
혜택 100% 받을 수 있어요. 놓치지 마세요. 지금 바로 신청하세요!
```
**기대**: A 매칭 (마케팅 키워드 2개+: 이벤트·혜택·받을 수 있어요·놓치지·신청하세요) → LMS 마케팅

## 회귀 신호 — 실패 시 즉시 캡처

| 케이스 | 기대 | 실패 신호 |
|---|---|---|
| V1~V5 | VOC 분류 | "유형 확인 부탁드립니다" → G 룰 미작동. 입력·출력 캡처 |
| L1~L4 | LMS 분류 | (광고) 없는데 마케팅으로 잘못 분류 / 케이스 6 발동 |
| M1·M2 | 마케팅 분류 | 일반 LMS로 잘못 분류 / 심의문구 ※ 3종 누락 |
| A1 | 알림톡 분류 | LMS로 잘못 분류 |
| S1 | SMS 분류 | LMS 5단 구조로 변환 (SMS는 단문 유지해야) |

## 추가 회귀 점검

### R1. 정답데이터처럼 교정되는지 — LN102 case
```
[미래에셋대우] 융자만기일 안내
#{고객명} 고객님
#{계좌번호}
#{종목명}의 융자 만기일을 알려드립니다.

융자일자 : #{YYYY/MM/DD}
만기일자 : #{YYYY/MM/DD}
융자수량 : #{대상수량}주
융자금액 : #{대상금액}원

온라인, 영업점 또는 고객센터를 통해 연장가능여부를 확인하신 후 만기연장을 신청하시거나 만기일까지 융자금을 모두 상환하여 주시기 바랍니다.

☎고객센터: 1588-6800
```

**기대 교정 결과** (정답데이터 verbatim):
```
[미래에셋증권] 융자 만기일 안내

#{고객명} 고객님, 아래 계좌의 융자 만기일을 안내해 드립니다.
온라인에서 만기일 연장 가능 여부를 확인하신 후, 만기일을 연장하거나 만기일까지 융자금을 모두 상환해 주세요.

■ 계좌정보
- 계좌번호: #{계좌번호}
- 종목명: #{종목명}

■ 융자만기정보
- 융자일: #{YYYY/MM/DD}
- 만기일: #{YYYY/MM/DD}
- 융자수량: #{대상수량}주
- 융자금액: #{대상금액}원

■ 꼭 확인해 주세요
* 만기일까지 융자금을 상환하지 않을 경우, 만기일 다음 영업일에 계좌 내 현금자산(외화예수금 포함)이 자동으로 우선 상환됩니다.

■ 문의
미래에셋증권 고객센터 1588-6800
```

**일치율 점검**:
- 회사명 통일 (대우 → 증권) ✓
- 도입 한 줄 요약 ✓
- ■ 블록 시퀀스 (계좌정보·융자만기정보·꼭확인·문의) ✓
- ☎ → ■ 문의 변환 ✓
- 한자어 순화 (만기연장을 신청하시거나 → 연장하거나) ✓

위 일치율이 80%+ 나오면 정답데이터 anchor 잘 작동. 70% 미만이면 RAG retrieval 실패 의심 → 추가 fix.
