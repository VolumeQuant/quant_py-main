## 2. 직원용 v9.4 → v11.0

### 핵심 변화

v9.4의 자체 룰 5종이 사용자 강조 "strict 가이드 준수" 원칙과 충돌. v11.0은 가이드 본문 변환 룰만 적용하고 자체 룰 모두 제거.

### 제거된 자체 룰 (5종)

| # | 자체 룰 | 제거 근거 |
|---|---|---|
| 1 | 인사·마무리 자동 추가 의무 | 가이드 명시 없음 |
| 2 | "안녕하십니까" 사용 | 가이드 67p, 74p 모두 "안녕하세요" (해요체) |
| 3 | 호칭+인사 결합 ("[호칭] 안녕하십니까.") | 가이드 명시 없음 |
| 4 | "고객" → "고객님" 자발 추가 | 가이드 36p는 CX 도메인 ("고객님" 첫 문장), 직원용 적용 X |
| 5 | (인사 없음) / (마무리 없음) 수정 사항 표기 | 가이드 명시 없는 자체 표기 |

### 가이드 명백 위반 수정 (2건)

| # | 수정 전 (가이드 위반) | 수정 후 |
|---|---|---|
| 1 | EX2 "송부 부탁드립니다" → "보내 주시기 바랍니다" (더 길어짐) | "보내 주세요" (가이드 29p 직접 인용 "~하여 주시기 바랍니다 → ~해 주세요") |
| 2 | EX5 "괜찮으시면 알려주세요" → "알려 주시기 바랍니다" (더 길어짐) | "알려 주세요" (가이드 29p) |

### 도메인 차이 명시 추가

CX 가이드에서 직원용 적용 X 항목 명시:
- 가이드 36p "첫 문장 '고객님' 시작" — 직원 쪽지는 호칭 그대로
- 가이드 61-62p LMS 5단 구조 — 직원 쪽지에 ■ 블록 X
- 가이드 67p 이메일 인사말 / 74p VOC 오프닝 — 다른 도메인
- 가이드 63-64p LMS 마케팅 심의문구 — 광고 도메인 X

### 적용 룰 15가지 (모두 가이드 Np 인용)

## 4. 공통 개선 (3개 시스템 적용)

### 출력 형식 1줄 압축 통일

```
## [수정 사항] (3건)

1. "익일" → "다음 날"  ·  한자어 순화 (가이드 134p)
2. "송부 부탁드립니다" → "보내 주세요"  ·  한자어 + 정중 어미 (가이드 29p, 134p)
3. "양지하시기 바랍니다" → "참고해 주세요"  ·  정중 어미 자연화 (가이드 26p)
```

- 마크다운 굵게(`**`) 제거 → 출력 토큰 ↓
- 가운뎃점(`·`) 시각 분리 → 스캔 가능
- 정답 12선 EX `변환>` 형식과 anchor 일치 → 회귀 차단

### 가이드 Np 의무 명시

모든 [수정 사항] 항목에 specific 가이드 페이지 인용. "가이드 해당 섹션" 같은 vague 표기 절대 금지.

### 사유 → 가이드 Np 매핑 표 추가

3개 시스템 모두 매핑 표로 일원화. 모델이 자체 페이지 추측·환각 차단.

### 가이드 9p 브랜드 보이스 4축 명시

Client First / Trustworthy / Contributive / Insightful — 모든 톤 룰의 상위 원칙으로 인용.

### "쓸데없이 정중 X" 룰 명시

가이드 9p Client First "과장된 표현이나 미사여구는 피합니다" + Trustworthy "불필요한 감정 표현은 절제합니다" 직접 인용.

## 8. 핵심 교훈 (인계 메모)

이전 작업자의 50회 시행착오에서 누적된 자체 룰이 가이드 미준수의 주된 원인. 모든 룰을 가이드 본문 직접 인용으로 검증해야 함. "기존 파일에 있던 룰이니까" 보존하면 잔재 누적 → 가이드 위반 누적.

**작성 원칙 (이후 버전 작업자용):**
- 모든 룰에 가이드 Np 또는 법령 조문 인용 의무
- "기존 룰 carry-over"는 가이드 검증 후에만 OK
- 자체 발명 룰은 즉시 식별 → 삭제 또는 "운영 보조 룰" 섹션으로 분리
- EX는 가이드 본문 사례를 verbatim 재현 우선
- byte 절감 < 출력 토큰 절감 (인사이트: 출력 토큰 길이가 글자 누락에 더 큰 영향)
