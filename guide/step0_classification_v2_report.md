# Step 0 유형 분류 룰 v2 — 검증 보고서

작성일: 2026-05-06
계기: 사용자 보고 — "유형 확인 부탁드립니다"가 무한 발동, 명확한 입력도 분류 못함. 특히 사용자 사례 "고객님 불편을 드려 거듭 사과드립니다..." (명백한 VOC) 분류 실패.

---

## 1. 근본 원인 진단

**v1 룰의 문제점**:
1. **VOC 시그널이 정답데이터에 없는 키워드 의존** — `VOC전담팀`/`VOC 담당자`/`VOC 파트` 정답2 dump에서 **0건**. 절대 매칭 안 됨
2. **2개+ 매칭 요구** — VOC 시그널 5개 중 2개+ 요구. 단순 사과·죄송 답변은 1개만 매칭 → 실패
3. **LMS 고지 strict 조건** — "■ 블록 2개 이상 + 본문 200자 이상" 요구. 짧은 LMS 안내·SMS 보더라인 입력 매칭 X
4. **fallback anchor "조금이라도 애매하면"** 너무 강함 → 케이스 6 무한 발동

## 2. 정답데이터 시그널 grep 통계 (정답2 dump 기준)

| 시그널 | 빈도 | 의미 |
|---|---|---|
| `[미래에셋증권]` 말머리 | **3,536건** | LMS·SMS·알림톡·이메일 거의 모두 |
| `(광고)` 마케팅 | **18건** | LMS 마케팅 명확 시그널 |
| `7일 동안/7일 내` | **347건** | 알림톡 만료 안내 |
| `놓치지 마세요/지금 바로/혜택/이벤트/쿠폰` | **290건** | 마케팅 톤 키워드 |
| `[연금인사이트]` 등 | **0건** | 뉴스레터 (가이드 backing only) |
| `미래에셋증권과 함께해 주셔서 감사합니다` | **2건** | 이메일 인사말 (가이드 67p) |
| `VOC전담팀/VOC 담당자` | **0건** | VOC (가이드 + 직원 wording 패턴 backing) |
| `고객님, 안녕하세요` | **2건** | VOC 시작 (드뭄) |
| `사과드립니다/죄송합니다/불편을 드린/양해` | **12건+** | VOC 핵심 wording |

**중대 발견**: VOC 시그널은 정답데이터로 추출 불가 (정답에 VOC 4건뿐). 가이드 + 직원 작성 wording 패턴 backing 필요.

## 3. 룰 v2 — 정답데이터 + 가이드 backing

```
[2단계 — 위에서 아래 순서로 검사, 매칭 시 즉시 채택]

A. LMS 마케팅 (가이드 63-64p, 정답2 18건)
   - "(광고)" 텍스트 또는
   - 마케팅 키워드 2개+ + 해요체

B. 뉴스레터 — [연금인사이트]/[라이프스타일]/[정보가치 공감]/[리서치] 말머리

C. 이메일 (가이드 67p) — "미래에셋증권과 함께해 주셔서 감사합니다." 또는
   메일 양식 + "이메일/메일/email" 키워드

D. 카카오 알림톡 (정답2 7일 만료 347건)
   - [미래에셋증권] + (CTA 버튼 또는 7일 만료 또는 알림톡 키워드)

E. SMS (가이드 59p) — [미래에셋증권] + 본문 ≤45자 + ■ 0개

F. LMS 고지/공지/안내 (정답2 3,536건)
   - A·B·C·D·E 미매칭 + [미래에셋증권] 또는 [미래에셋대우] 말머리

G. VOC 답변 (가이드 123p)
   - [미래에셋증권] 0개 + ■ 0개 + 다음 1개+
     · 사과/죄송/불편 (사과드립니다·죄송합니다·죄송·불편을 드린·양해)
     · 공감/재발 방지 (공감·이해·재발 방지·개선·강화·점검)
     · VOC 도메인 (민원·콜센터·콜백·VOC·응대 품질)

[3단계] 위 A~G 모두 미매칭 → 케이스 6
```

## 4. 시뮬레이션 검증 — 8 case 분류 결과

| # | 입력 사례 | 매칭 룰 | 결과 |
|---|---|---|---|
| 1 | "고객님 불편을 드려 거듭 사과드립니다. 대단히 죄송합니다. 진심으로 사과드립니다." | G (불편·사과·죄송 매칭, 말머리 0, ■ 0) | **VOC** ✓ |
| 2 | "(광고)[미래에셋증권] 계좌 개설 혜택..." | A ((광고) 매칭) | **LMS 마케팅** ✓ |
| 3 | "[미래에셋증권] 융자 만기일 안내\n#{고객명} 고객님...\n■ 계좌정보..." | F ([미래에셋증권] 매칭) | **LMS 고지** ✓ |
| 4 | "[미래에셋대우] 융자만기일 안내..." | F ([미래에셋대우] 매칭) | **LMS 고지** ✓ |
| 5 | "[미래에셋증권] 신청내역\n#{URL}\n* 링크는 7일 동안 확인할 수 있습니다." | D ([미래에셋증권] + 7일 만료) | **알림톡** ✓ |
| 6 | "[미래에셋증권] 매매거래 정상 처리" (45자 이내) | E ([미래에셋증권] + 45자 + ■ 0) | **SMS** ✓ |
| 7 | "안녕하세요, 김미래 고객님. 콜센터 응대로 불편을 드려 사과드립니다." | G (콜센터·불편·사과 매칭) | **VOC** ✓ |
| 8 | "[연금인사이트] 5월 시장 전망..." | B (뉴스레터 말머리) | **뉴스레터** ✓ |

**8/8 통과 — 모든 case 정확 분류**.

## 5. 우선순위 일관성 검증

같은 입력은 항상 같은 결과:
- (광고) [미래에셋증권] 입력 → A 먼저 매칭 → **항상 LMS 마케팅** (F는 검사 X)
- [미래에셋증권] + 7일 만료 입력 → A·B·C 미매칭 → D 매칭 → **항상 알림톡**
- [미래에셋증권] + 일반 본문 → A·B·C·D·E 미매칭 → F 매칭 → **항상 LMS 고지**

★ "위에서 아래 순서로 검사, 매칭 시 즉시 채택" 명시 → 일관성 보장.
