# Mi-Tone v5.21 변경 안내 (현업용 1페이지)

> 메시지 작성 직원이 알아야 할 변경점만. 1페이지로 간결하게.
>
> ⚠️ **사용자 검증 필요**: 부서 담당자 이름·내선·연락처는 회사 실제 정보로 채워 넣은 뒤 사용. `[…]` placeholder 영역 모두 검증.

## 🎯 무엇이 달라졌나요?

### 1. **말투 자동 분기** (가장 큰 변화)

이전(v5.20): 모든 메시지가 "~합니다" 격식체로만 변환
**변경(v5.21)**: 메시지 종류·위치별 자동 분기

| 메시지 종류·위치 | 말투 |
|---|---|
| 공지/고지/안내 본문 | ~합니다 (그대로) |
| **행동 요청** ("확인해 주세요" 등) | **해요체** ★변경 |
| 항목명·기준값 | 명사형 |
| **마케팅 카피·이벤트 인사말** | **해요체 + 부드러운 표현** ★변경 |

→ 이제 마케팅 메시지에 "혜택이 기다리고 있어요~", "지금 신청해 주세요" 같은 친근한 표현이 자연스럽게 나옵니다.

### 2. **시스템 행위는 그대로** (피동 유지)

이전(v5.20): "주문이 체결되었습니다" → "주문을 체결했습니다"로 변환됨
**변경(v5.21)**: "체결되었습니다" 그대로 유지

→ 시스템(거래소·증권사)이 한 행위는 피동 표현이 자연스러우니 변환 안 합니다.
- ⭕ "주문이 체결되었습니다", "이체되었습니다", "처리되었습니다"
- ❌ "주문을 체결했습니다" (어색)

단 고객이 한 행위는 능동 그대로:
- ⭕ "신청하셨습니다", "등록하셨습니다"

### 3. **VOC 답변 시나리오 자동 분기** (신규)

VOC 답변 4가지 상황을 자동 인식해서 다르게 답변:

1. **일반** (콜센터 짜증 등 회사 귀책 X)
2. **처리 불가**
3. **사실 다른 주장** ("~인 줄 알았는데")
4. **답변 초안 편집** (외부 위험 요소 삭제 — 금감원 민원·언론 제보·법적 조치 자동 삭제)

### 4. **검수 통과 정답지 220개 학습** (Mi-Tone 진화)

회사 공식 1차·2차 메시지 개선 약 220건의 검수 통과 사례를 Mi-Tone이 학습했습니다. 같은 패턴 메시지가 들어오면 정답에 가까운 형태로 교정됩니다.

## ⚠️ 변하지 않은 것 (그대로)

- 숫자·금액·날짜·계좌번호·법령명·플레이스홀더 `#{고객명}` 등 **사실 데이터는 한 글자도 안 바뀜**
- LMS 5단 구조 (말머리·제목·인사말·본문·문의)
- ☎ 자동 삭제 + ■ 문의 블록 자동 추가
- 광고 규제 자동 점검 (수익률 단정 등)
- 한자어 순화 (당사 → 미래에셋증권)

## 🚨 알아두세요

### 글자 누락(탈자) 별도 점검 트랙
시스템이 가끔 "랩어카운트"를 "랩어카운"으로 글자를 빼먹는 경우가 있습니다. v5.22에서 별도 후처리 검증기로 보강 예정. 그동안 **출력 결과는 발송 전 한 번 더 확인**해 주세요.

### 검수 미진행 메시지는 그대로
정답데이터에 없는 새로운 메시지는 가이드 룰 기반으로만 교정됩니다. 더 자연스러운 검수 형태가 필요하면 금융소비자보호팀 검수를 받아주세요.
