### 메타 해설 docx의 RAG 오인 위험
docx에 "v5.21 다이어트·카드 RAG·톤 매트릭스·tie-breaker·5 케이스" 같은 Mi-Tone 내부 용어 박히면 RAG 검색 시 LLM이 가이드 룰로 오인. **메타 해설은 RAG가 아니라 시스템 프롬프트에 들어가야 함**.

### 결정 의존 관계 그래프 (v5.21 사례)
- C 카드 양식 → A 5·6 (tie-breaker·카드 마커) → E 도구
- A 1 페르소나 (독립)
- A 2 톤 매트릭스 (독립)
- B 신규 4 docx (각 독립, 항목별 차등)

- D 거버넌스 (운영 영향 0, 후순위)
**의존 관계대로 결정 순서 안 잡으면 후속 결정에서 역산 필요**

**v3 마커 의미 정정 (★중요★)**:
- 이전 (잘못): "재개선 진행 중·인용 금지·방향성만 참조"
- 정확 (xlsx 2행 메모 직접 인용): "현업 피드백에서 다시 개선(또는 와이어링크 추가 개선 의견)" → **외주업체 추가 개선 + 다영 2교 = 가이드 부합도 더 높음, 학습 자료**

**Why**: 사용자 통찰("귀중한 정답데이터 — 모두 학습 자료") + 김보민 쪽지("약 500건 개선안 학습 부탁") + xlsx 메타·숨김 시트 검증으로 정답데이터2 521건 모두 카드화 정당. EVAL_ONLY 라벨링은 잘못된 가공.

**How to apply**: 정답데이터3·4 발간 시 동일 절차 — (1) xlsx 메타 확인 (2) 숨김 시트 점검 (3) 2행 메모 + 셀 코멘트 (4) 시트별 컬럼 매핑 정확히 (5) TOBE 채워진 행 모두 카드화 (NEGATIVE만 분리).

### "근거 있는 자의 vs 근거 없는 자의" 분리
모든 자의가 "폐기"가 아님. 인라인 마커 `<v2>`/`<v3>`처럼 데이터 무결성(rich text 색 보존) 측면 명확한 이유 있는 자의도 있음. 결정 옵션 제시 시 라벨 분리 필수

### 메모리도 환각 가능 (인사이트 ⑮)
메모리는 사용자 정독 결과라 신뢰도 높지만 자의 명칭이 누적될 수 있음. PDF 정독 검증으로 발견:
- "16 룰" → 가이드 PDF 0건. 정확 표기는 "핵심원칙 4 / 세부항목 16 / 세부원칙(설명)"
- "5단 문체 혼합" → 가이드 26p 0건. 실제는 "다양한 문체(하십시오체+해요체+명사형 3종) 활용"
- 메시지 유형 10p 표에 VOC 행 없음 (VOC는 67p+ Part 3 별도 섹션, 10p+11p 짝표가 정확)
- **적용**: 메모리 결정안 채택 시 PDF 본문 직접 추출 검증 의무. 메모리 갱신 시 자의 명칭 사용 금지, PDF 본문 표기 그대로

### 가이드 본문 인용도 컨텍스트 검증 필수 (인사이트 ⑪)
가이드 119p 페르소나 "브랜드 마케터·카피라이터"는 PDF 본문 직접 인용이지만 ChatGPT 사용자용 프롬프트 페르소나 예시 — Mi-Tone 시스템 자체의 페르소나가 아님. **표면 인용(단어가 PDF에 있음) ≠ 컨텍스트 적용**. 인용 추가 시 (a) 단어 PDF 등장 확인 (b) 인용 컨텍스트가 Mi-Tone 운영 위상에 맞는지 추가 검증

### v5.20 베이스도 가이드 미근거 인용 누적 (인사이트 ⑬)
v5.20 = 30번 시행착오 결정체이지만 일부 가이드 미근거 인용 누적 (예: 자본시장법 제57조 26회). 룰 7 "다이어트 = 빼도 될 근거 명확한 것만"의 한계: v5.20 자체도 부분적으로 빼야 할 항목 포함 가능. **v5.20 인용도 항목별 가이드 PDF 검증 필요**

### "가이드 미근거 + 정답데이터 반복 = 운영 룰" 예외 원칙 (인사이트 ㉓ — ★중요★)
룰 3·4 ("가이드 본문 미근거 표기 금지")의 핵심 예외:
- **가이드 본문 미근거 + 정답데이터1·2에 반복·최종 결정 포함 → 운영 룰로 채택**
- 정답데이터는 회사 공식 검수 통과 형태 → 가이드 본문 외 회사 운영상 실제 적용 룰

## 룰 9: 가이드 우선 + 정답데이터 보수적 채택 기준 (★중요★)

인사이트 ㉓ 보강 — 단순 "정답데이터 등장 = 채택" 아님. 잘못된 원본 반복 방지.

**판단 절차**:
| 단계 | 판정 | 채택 여부 |
|---|---|---|
| 1 | 가이드 본문 PDF 명시 | ✅ 채택 (1순위, 룰 3 본래) |
| 2 | 가이드 미명시 + TOBE(검수자 개선안) 빈도 5회 이상 | ✅ 운영 룰 채택 가능 |
| 3 | ASIS만 등장 (잘못된 원본일 수 있음) | ❌ 채택 X |
| 4 | TOBE 빈도 5회 미만 (1·2회) | ❌ 자의 추가 의심 |
| 5 | 가이드 vs 정답데이터 충돌 | 가이드 우선 (단 빈도 매우 높을 때 사례별 검토) |

**보수적 빈도 기준**: TOBE 5회 이상 = 525 TOBE cells의 약 1% (통계적 의미 있는 최소선)

**v5.21 실증 적용**:
- 자본시장법 제57조 26회 (광고 규제) — 정답데이터 광고 메시지 필수 → 유지 ✅
- 시스템 피동 OK 화이트리스트 8개 → **4개로 축소** (체결 99p 명시 + 취소 20·접수 13·처리 7회). 발송 1·이체·정정·진행되지 0회는 미채택
- "등록되었습니다" 17회 (피동 유지) — v5.20 룰 "등록하셨습니다" 능동 변환과 충돌 → tie-breaker 규칙 1로 카드 우선
- VOC_misclaim 트리거 키워드 (~라고 알고 등) 0회 → 자의 보조 신호 라벨 + LLM 의미 식별 우선

**Why**: 가이드 = 회사 표준의 일부. 정답데이터 ASIS = 잘못된 원본 가능. 정답데이터 TOBE 빈도 ≥5회 = 검수자 다수 통과 = 운영 룰. 보수적 기준 없으면 잘못된 원칙 반복 시스템 위험.

**How to apply**: 가이드 미근거 항목 발견 시 (1) 정답데이터 TOBE 빈도 검증 (2) 5회 이상 + 가이드 99p 같은 일반 룰 근거 있으면 채택 (3) 빈도 미달은 자의 확장 라벨 + 일반 룰만 LLM에 박아 의미 판단 위임. 룰 9 자동 점검 도구: `output/도구/validate_insight23.py`

**Why**: 사용자 진급 직결 작업. v5.21 1차 시도에서 망친 8 패턴 반복하면 신뢰 회복 불가

**How to apply**: v5.21 2차 시도(또는 v5.22 이후) 작업 시작 시 이 메모리 무조건 읽고 룰 1~8 모두 점검 후 진행. 룰 위반 발견 즉시 작업 중단·보고
