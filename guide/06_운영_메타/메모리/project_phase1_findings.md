미래에셋증권 「고객중심 언어 가이드」 안내 PDF(6p) + 본문 PDF(147p) 전체 정독으로 도출한 핵심 발견. v5.21 설계와 정답데이터2 통합 전략의 근거.

## 5대 결정적 발견 (북극성 관점)

1. **시스템 위상**: 회사 공식 향후 일정에 명시("고객중심 언어AI 개발"). 정답데이터1·2 = 회사 공식 1차·2차 메시지 개선 사례 (Phase G EDA 결과: 정답데이터1 = 메시지검토 시트 30건 / 정답데이터2 = 4 시트 TOBE 채워진 521건). "현업 작성 → 금융소비자보호팀 검수" 사이의 사전 가공 도구.

2. **★해요체 충돌★**: 가이드 15+ 페이지에서 "요청·권유·제안·마케팅 = 해요체" 명시. 마케팅은 직접 "친근한 해요체" 명시(p63). v5.20 line 9 `해요체 혼용 금지`는 정면 위반. 정답데이터2 미수용 사유 1순위 후보.

3. **Part 4 = LLM 메타 가이드**: p119 회사 공식 권장 LLM 프롬프트 구조 = 페르소나 + 정보·작성조건·제한사항 + 예시. p120 "**과도한 장황한 구체성이 LLM 출력을 망친다**" 직접 경고. v5.20 840줄은 이 경고에 정확히 해당.

4. **3개 결정 매트릭스**: ① 메시지 유형 × 4종 보이스(Main/Sub) — p10 (공지성·고지성·마케팅 3행, VOC 행 없음. 11p에 프로세스 유형별 4행 짝표 별도) ② Part 2 핵심원칙 4 × 세부항목 16 — p35 한 장 요약 + 36-57p 각 항목 상세 ("16 룰"이라는 단어는 가이드 PDF 0건, 자의 명칭 금지) ③ 세대 × Do·Don't — p39-40 5컬럼(특징·필요한 톤·Do·Don't·Life Stage). v5.20의 흩어진 룰 다수를 이 3 표로 압축 가능.

5. **사례 의존 패턴**: 정당화 prefix + 안심 정보 / VOC 4 시나리오 / 표준 문구 풀 / 강조 재구성 / 전문용어 풀이 — 룰로 표현 어렵고 골든 사례 학습이 정답. **정답데이터2를 이런 카드 형태로 가공해 RAG에 넣는 것이 북극성과 정확히 일치**.

**Why**: Phase 1B의 5대 발견이 v5.21 설계 방향과 정답데이터2 통합 전략의 근거. 사용자가 "기계적·어색"으로 느끼는 원인의 운영적 정의 = "v5.20이 회사 공식 가이드의 룰·톤·구조와 충돌하고, 정답데이터(검수 통과 형태)와 거리가 멀다".

**How to apply**: v5.21 설계 시 (a) 룰 추가 X, 압축·분담 ✅ (b) 정답데이터2 → 골든 카드 형태로 RAG (c) 페르소나 = "검수자".

## 충돌 30+ 항목 → 5 카테고리 분류

**A. 압축 매트릭스 (3 표로 통합)**: 메시지 유형×보이스 / Part 2 16 룰 / 세대×Do·Don't

**B. 결정성 표 직박**: 통일 용어 14 / 한자어 28 / 외래어 13 / 심리 불편 12 / 유의어 7쌍 / 문장부호 12 / 광고 심의문구 3종

**C. RAG 골든 카드 (★정답데이터2 활용 핵심★)**: 정당화 prefix+안심 정보 / VOC 4 시나리오 / 메시지 유형별 표준 문구 풀 / 강조 재구성 / 전문용어 37개

**D. 페르소나로 압축**: "검수자 페르소나 + 편집정책 가치(공정·정확·독립)"

**E. 신규 1줄 룰**: 사물 주체 -시- / "~들" 복수형 / 시스템 주체 피동 OK 예외 / 컨펌 셰이밍 회피 / 채널 키워드 일관성
