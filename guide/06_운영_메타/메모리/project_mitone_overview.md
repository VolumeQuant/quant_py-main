미래에셋증권 사내 RAG 시스템 Mi-Tone 개발 프로젝트. CX·UX 두 시스템(C:\dev\guide).

**시스템의 회사 내 위상**
- 회사 공식 향후 일정에 명시: "고객중심 언어AI 개발 (상반기 중) — 언어가이드 자동반영 AI개발"
- 가이드 발간 주체: 금융소비자보호팀(CX 클라이언트, 우선순위 높음) × 디지털마케팅팀(UX) × 프로세스운영팀
- 운영 흐름: 현업부서 작성 → 금융소비자보호팀 검수 → 프로세스혁신본부 코드 부여(MAPIS 3283 등록). 시스템은 현업이 작성 단계에서 사용해 검수 통과 형태로 미리 가공하는 위치.

**Why**: 사용자 진급에 큰 영향이 있는 매우 중요한 작업. 2,500명 이상 직원이 고객 메시지 작성 시 사용하게 됨 → 회사 신뢰도와 직결.

**How to apply**:
- "사람처럼 잘해주는 시스템"의 운영적 정의 = "금융소비자보호팀 검수를 한 번에 통과하는 형태"
- 정답데이터 = 검수·결제라인 통과한 공식 표준이지 단순 사례 아님

**데이터 구조 (xlsx 직접 검증 결과 — 메모리 ㉕·㉚·㉛·㉜ 적용)**

**사용자가 만질 수 있는 제어 범위 (제약 조건)**
1. vector DB에 들어가는 docx 문서
2. 시스템 프롬프트 파일 (v5.20 CX 840줄 / UX 571줄)
