이 사용자와 Mi-Tone 관련 분석·기획 작업을 할 때는 다음 방식으로 진행한다.

1. **단순 요약 금지** — 분석을 통한 인사이트·패턴·추세 도출이 매 단계의 결과물.
2. **단계별 누적** — 각 단계 끝에 인사이트를 명시적으로 정리하고, 다음 단계 들어갈 때 그 인사이트를 활용. divide-and-conquer.
3. **북극성**: "정답데이터2를 효과적으로 녹여 현업이 '기계적이 아니라 사람처럼 교정해준다'고 느끼는 시스템". 모든 분석·결정은 이 기준으로 평가.
4. **시간 제약 없음** — 깊이 우선. 모든 문서 한 줄씩 정독, 빠른 정리·통과 금지.
5. **순서**: 의도 → 본문 → 현재 구현 → 신규 데이터. 즉 (1) 가이드 안내 PDF (2) 가이드 본문 PDF (3) RAG docx (4) 시스템 프롬프트 (5) 정답데이터1·골든 (6) 정답데이터2.
6. **사용자 승인 없이 RAG에 새 문서를 넣거나 시스템 프롬프트를 임의 수정하지 않는다** — 모든 변경은 분석·계획 단계 완료 후 사용자 컨펌 거침.

**Why**: 사용자가 첫 단계에서 "그냥 1차처럼 골든 docx 만들어 넣으면 되나" 식의 단순 결론을 거부하고, 본문부터 한 줄씩 다시 읽으라고 명시함. 분석 깊이가 결과물의 질을 결정한다는 강한 신념.

**How to apply**:
- 각 phase 끝에 "페이지/파일별 발견 → 북극성 관점 인사이트 N개 → 다음 단계 영향" 3블록으로 정리.
- 사용자가 "진행해" 라고 명시하면 다음 phase로 자동 진행, 아니면 매 phase 끝에 짧게 다음 단계 시작 의사 확인.
