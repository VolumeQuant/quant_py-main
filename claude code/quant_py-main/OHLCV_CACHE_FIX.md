## 1. 발견된 버그 3개

### 버그 1: OHLCV 증분 업데이트 오염 (커밋 84ab431)

- 원래 유니버스 종목(~880)만 캐시해야 하는데 전 종목이 들어감
- **수정**: `keep_tickers = cached_tickers | universe_set` 필터링

### 버그 2: KRX 인증 고장 (2주간)
- 비밀번호 변경 후 config.py 미반영 + 중복 로그인 미처리
- **수정**: 비밀번호 업데이트 + krx_auth.py CD011 처리

### 버그 3: 거래대금 1일 래그 (이번에 발견)

## 3. 왜 universe/scored가 크게 늘었나

**원본 시스템의 한계:**
- 최초 실행(2/5) 시 OHLCV 캐시에 94종목만 존재
- 이후 증분으로 늘어나지만, 매일 새로 들어오는 종목만 추가
- 유니버스에 있지만 OHLCV 캐시에 없는 종목 → MA120 필터에서 스킵 → scored에서 제외
- 결과: scored가 실제보다 적게 나옴 (100~183개)

**재계산:**
- 각 날짜의 전체 유니버스에 대해 OHLCV 수집
- 모든 유니버스 종목이 MA120 필터를 거침
- 결과: scored가 정확하게 계산됨 (252~450개)

**거래대금 래그 수정 효과:**
- 1일 래그 제거 + market_cap 파일 직접 사용
- 유니버스가 ~100~170개 더 정확하게 산출
