# 국면 조기위험탐지 — 종합 검증 결과 (2026-06-20, 자율주행)

## 한 줄 결론
**조기방어 신호는 전부 실패한다.** 멀티에이전트 리서치(61후보)에서 추린 모든 구현가능 후보를
down-only BT에 적용한 결과, baseline(MA20/80/5, Calmar 4.05/MDD 25.9%)을 이기는 게 하나도 없음.
약세장 MDD는 줄여도 강세장 수익을 더 많이 포기하거나, 집중도 증가로 MDD가 오히려 악화.
**유일하게 작동하는 리스크 레버는 게이트가 아니라 현금버퍼(메타 배분).**
단 ★브레드스는 훌륭한 **진단지표**(현재 b200=21% vs 평균39% = 광범위 약세 실재, 사용자 직감 맞음).

## 방법
- 리서치: Workflow 멀티에이전트 8각도 병렬(논문/GitHub/웹) → 61후보. 막판 spend-limit로 중단,
  트랜스크립트서 회수(`research/_ew_recovered.json`). 최대테마 breadth(21)·ML-changepoint(10)·credit(8).
- BT: 각 후보를 production down-only state에 게이트/overlay로 적용. baseline 대비 Calmar/MDD/약세장/현금일수.
  내 시뮬은 고정3슬롯=Calmar 4.077/MDD 25.9%로 공식 baseline 재현 검증 완료.

## ★왜 전부 실패하는가 (근본 원인)
1. **우리 엣지 = 성장 리더에 공격적 참여.** "시장 약세시 방어" 신호는 *우리 리더가 이기는 구간*에
   우릴 빼내 → 엣지 자체를 희생. 모든 defend-on-weakness가 이 함정.
2. **약세는 우리가 안 사는 종목에 있다(사용자 직감).** 브레드스 약세=소형주. 우리는 시총≥1000억+
   MA120위 리더만 보유. 브레드스로 게이트하면 *남의 약세로 우리가 벌받음*.
3. **MA20/80은 Goldilocks.** 늦어서(-23%) 휩쏘 안 당하고, 그래도 *연장* 약세는 회피
   (코로나 크로스후 -26.6%·2021크로스후 -25.4% 추가하락을 현금으로 피함). 더 빠르면 휩쏘로 자멸.
4. **3슬롯 분산이 곧 포지션 리스크컨트롤.** 줄이면(soft) 집중도↑로 MDD 악화 = 역효과.
5. **-23% 후행은 '휩쏘 안 당하는 값'.** 추세게이트의 불가피한 특성. 개별 SL/TS도 과거 기각(수익악화).
   → 시장레벨·포지션레벨 조기방어 *둘 다* 실패 입증됨.

## 실전 함의 (바꿀 것 없음 + 형 직감 활용법)
- **게이트·슬롯·exit 전부 현행 유지** (조기방어 도입 안 함). 데이터로 결판.
- **유일한 '덜 잃기' 레버 = 현금버퍼(메타, 사용자 영역).** 70/30→MDD~19%, Calmar~4.1 유지(휩쏘0).
  불안하면 시스템 비중 축소(90/10→80/20). 게이트를 만지는 게 아님.
- ★**브레드스를 표시전용 진단지표로 추가 권장**: b200(%>MA200), HL(신고가-신저가). 매매엔 안 엮고
  "내 시스템(리더) vs 시장폭" 상황인지용. 현재 b200=21%=소형주 광범위약세(사용자 체감과 일치),
  단 우리 보유리더는 강세(코스피·유니버스EW=boost) → 노출 적음을 수치로 보여줌.

## 현금버퍼 프론티어 (유일하게 작동하는 '덜 잃기' 레버 — 정밀수치)
메타배분 BT(기존 문서화, 7.4년)로 이미 정밀: 시스템:현금 블렌드(현금 연3%)
- 100:0 = Calmar 4.05 / MDD 25.9% (현행 100% 시스템)
- 90:10 ≈ MDD ~23% / Calmar ~4.1 (공격형)
- 80:20 ≈ MDD ~21% / Calmar ~4.1
- **70:30 = Calmar 4.14 / MDD 19.2%** (분기리밸·밴드, 보수형)
→ 현금버퍼는 MDD를 비례 축소하되 Calmar 유지(휩쏘0). "디바이스 같은 개별 아픔"이 부담되면
  게이트가 아니라 **시스템 비중을 낮춰라**(예 100→80:20). 조기방어 게이트는 전부 이보다 나쁨(위 표).
