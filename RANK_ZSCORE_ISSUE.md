# rank z-score 유니버스 불안정 분석 (2026-04-09)

## 현상
삼성전자가 하루 만에 composite_rank 11위→58위. 자체 z-score 값 거의 변화 없음.

## 직접 원인
run_daily가 매일 DART 갱신. 4/9에 31건 수집, 29건이 유니버스 내 종목.
이 29종목의 Growth TTM이 새 분기보고서로 변경됨.

rank z-score는 상대 순위 기반. 300종목을 1~300위로 줄세워 정규분포 변환.
29종목 Growth 원시값 변경 → rank 전체 재배열 → 삼성전자 앞으로 47개 종목 진입.
Growth가 공격 모드 65% 비중이라 composite rank 영향 극대.

## BT에서 동일하게 발생 (정상 동작)
- BT도 point-in-time rcept_dt 필터링 적용
- 매 BT 날짜마다 `eff_date <= date_ts` 조건으로 해당 시점 공시만 사용
- **BT와 프로덕션이 같은 방식으로 동작 → Calmar 6.62에 이미 반영됨**
- 버그가 아니라 시스템의 정상 동작

## CSWZ(횡단면 윈조라이즈 z-score) 검토 결과: 부적합

Growth만 raw z-score로 바꾸면 안정성이 개선될 것으로 예상했으나,
**공시 시즌 방향성 편향**(실적 좋은 기업이 먼저 공시)에서 오히려 악화:

| 시나리오 | Rank z-score | CSWZ | 결과 |
|----------|---|---|---|
| 랜덤 갱신 | ~4칸 | ~2칸 | CSWZ 2x 유리 |
| 상위 50% 편향 | ~8칸 | ~14칸 | Rank z 2.3x 유리 |
| 상위 25% 편향 | ~11칸 | ~28칸 | Rank z 2.5x 유리 |
| 상위 10% 편향 | ~17칸 | ~45칸 | Rank z 2.6x 유리 |

CSWZ는 방향성 편향에서 cross-sectional mean이 끌려가면서 불변 종목의 z-score가 급락.
Rank z-score는 순위 재배열만 하므로 방향성 편향에 더 강건.
