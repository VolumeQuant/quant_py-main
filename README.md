## 1. 핵심 아이디어

한국 시장에서 오래 관찰되는 패턴 하나에 집중합니다: **실적(매출·영업현금흐름·매출총이익)이
가속하는 중소형 리더는 주가가 그 정보를 한 번에 반영하지 않고 수개월에 걸쳐 따라간다.**
그래서 순위의 55%를 성장 팩터에 싣고, 모멘텀(30%)으로 "시장이 이미 알아채기 시작했는가"를
확인하며, 밸류(15%)와 과열 캡으로 추격매수만 걸러냅니다.

이 엣지의 성격을 숫자로 말하면 — 7.4년 백테스트에서 **수익의 75%가 상위 5% 거래에서**
나왔습니다(승률 53%, 손익비 3.8). 즉 대박 승자 몇 개가 전부입니다. 이 시스템의 모든 필터와
룰이 "승자를 죽이지 않는가"라는 관문(LOWO, [7절](#7-검증-철학))을 통과해야 하는 이유입니다.

## 2. 전략 규칙 (현재 운영본)

### 유니버스 → 필터 → 점수 → 순위 → 포트폴리오

```
┌─ 유니버스 (~2,700종목)
│    시총 1,000억+ · 거래대금(대형 50억/중소형 20억+) · 우선주/외국계 제외
│    금융·지주 제외 · 상장 6개월 미만 제외(모멘텀 계산 불가)
│
├─ 품질 관문 (하나라도 걸리면 탈락)
│    ① DART 분기보고서 8개(2년)+ — 성장 팩터를 믿을 수 있는 종목만
│    ② 4팩터 중 하나라도 -1.5σ 미만이면 제외 — 한 축이 극단 결함이면 다른 축이 좋아도 함정
│
├─ 점수 (공격 모드)
│    성장 0.55 + 모멘텀(12m) 0.30 + 밸류 0.15
│    + 단기모멘텀 0.05 + 저변동성 0.06     ← 가격 변동이 순위에 매일 반영되도록
│    + min(과열z, 0) × 0.2                ← 실적 대비 비싸진 종목만 감점 (싼 건 무보상)
│    − 함정 페널티                         ← 4절: 일회성 매출/이익, 최근 증자 등
│
├─ 순위: 3일 가중(0.4/0.35/0.25) weighted_rank + 3일 연속 상위권 ✅ 검증
│
└─ 포트폴리오: 상위 3종목 · 균등비중 · 매일 재심사
     이탈: 순위 5위 밖 or −15% 손절 → 매도 후 10거래일 재진입 금지
```

## 3. 왜 점수가 아니라 순위, 하루가 아니라 3일인가

**순위인 이유**: 한국 시장 ~1,900종목의 팩터 점수는 표준편차 노이즈가 큽니다. "점수 1.85 vs
1.79"는 정보가 아니지만 "1위 vs 7위"는 정보입니다. 점수 기반 매매는 백테스트에서 순위 기반에
일관되게 밀렸습니다 (Calmar 2.62 vs 3.39).

**3일인 이유**: 당일 순위(cr)는 하루짜리 노이즈에 흔들립니다. 그래서 매매 판단은 전부
3일 가중 순위(wr)로 합니다:

```
wr = 오늘 순위×0.4 + 어제×0.35 + 그제×0.25
예) 오늘 2위·어제 1위·그제 3위 → wr = 0.8 + 0.35 + 0.75 = 1.9 → "사실상 2위"
```

여기에 **✅ 검증**(3일 연속 상위 30위 내)까지 통과해야 매수 후보가 됩니다. 하루 반짝
스파이크는 ✅를 못 받고, 진짜 리더는 3일을 기다려도 그 자리에 있습니다.

## 4. 함정 방어 — 실계좌 손실에서 역설계한 필터들

이 시스템의 함정 필터는 이론이 아니라 **실제로 돈을 잃은 종목들**에서 나왔습니다.
공통 교훈: 잡을 수 있는 함정은 **구조적 회계 착시**뿐이고, 가격·거래량으로 함정을
구별하려는 시도(창의 피처 20+종 전수 검증)는 전부 실패했습니다 — 함정과 대박 승자는
진입 시점 가격 신호가 똑같기 때문입니다.

| 필터 | 실패 사례 (계기) | 잡는 방법 |
|---|---|---|
| **일회성 매출** (lumpiness) | 디바이스이엔지 — 분기매출 53→262→86→392억. 대형 수주가 터진 분기만 반짝인 프로젝트 회사를 시스템이 "지속 성장"으로 오인해 3위 추천, 다음 분기 꺼지며 7일 −50% | 최근 4분기 매출의 min/max < 0.25 (가장 약한 분기가 가장 센 분기의 ¼ 미만 = 어느 분기가 텅 빈 회사) → 성장점수 ×0.3 |
| **일회성 이익** (accruals) | 삼지전자·에스에이엠티 — 한 분기 일회성 이익으로 TTM이 뻥튀기, 시스템 1위까지 상승 | 이익은 나는데 현금이 안 들어오고(B>25) 한 분기에 쏠림(C>0.7) → 성장점수 ×0.24 |
| **최근 증자/분할** | 무상증자·유상증자 직후 종목 코호트는 이후 1년 −13% (7.4년 555건 실측) | 126일 내 하락형 자본변동 있으면 점수 −0.3 |
| **얇은 밸류트랩** | 대덕 — PER 184인데 PBR 1.08이라 "싸 보여서" 3위 진입. 이익이 없는데 장부가만 싼 지주형 트랩 | PER>40 & PBR<1.5 & 거래대금<50억 → 제외 |

반대로 **일부러 안 막는 것**: 고PBR 모멘텀 폭등주(제주반도체·브이엠 유형)는 필터로 거르면
승자까지 학살된다는 게 반복 확인됐습니다. 이런 종목의 리스크는 필터가 아니라 구조(3슬롯
분산 + 순위 이탈 + 손절)가 담당합니다.

## 5. 방어 구조 — 3층

| 층 | 감시 대상 | 신호 | 발동 시 |
|---|---|---|---|
| **시장 전체** | KOSPI MA20 vs MA80 | 데드크로스 5일 연속 확인 | **전량 현금.** 2000년 이후 6개 약세장에서 매수보유 대비 MDD −7~32%p 방어 (2008년 −32%p가 압권) |
| **시장 폭** | 23개 섹터 중 200일선 위 비율 | <35%가 3일 지속 | 노출 50% 축소 권고 — 지수는 메가캡이 떠받치는데 시장 내부가 무너지는 "느린 약세" 전용 |
| **개별 종목** | 보유 종목 순위·가격 | 순위 5위 밖 or 진입가 −15% | 즉시 매도 + 10거래일 재진입 금지 |

**손절이 −15%로 느슨한 이유**: 타이트한 손절(−8/−10%)은 백테스트에서 일관되게 손해였습니다
— 승자의 정상적 출렁임까지 잘라내기 때문. −15%는 "비용 0인 테일 보험"으로, 전체 성과를
건드리지 않으면서 최악 단일거래를 −35%→−20%로 절단합니다.

**재진입을 10일 막는 이유**: 이탈 후 4~10일 내 되사기 코호트는 승률 29%·평균 −2.05%
(신규 진입 +11.3%와 대비). 손절 후 반등도 실측상 대부분 데드캣입니다 — 3~5일 중앙 +5~7%
반짝 후 20일이면 중앙 −7.5%로 도로 무너집니다. 진짜 재가속(한 달 뒤 재복귀 코호트 +27.7%)은
10일 창과 겹치지 않아 놓치지 않습니다.

**조기경보를 일부러 안 다는 이유**: 8가지 조기위험탐지(빠른 MA, 드로다운 브레이커,
변동성 게이트 등)를 전부 측정했지만 baseline을 이기는 것이 하나도 없었습니다 — 시스템 안의
조기경보는 휩쏘 비용이 보호 이득을 초과합니다. 조기방어가 필요하면 시스템 밖 현금 버퍼가
맞는 레버입니다.

## 6. 시스템 아키텍처

```
평일 16:00 (Windows Task Scheduler) — run_daily.py
  ├ git pull                          다른 PC 변경 자동 반영
  ├ 거래일 판정                        pykrx 빈응답 시 yfinance 교차확인 (무소음 결번 방지)
  ├ data_refresher                    시총/재무(DART 증분)/수정주가/섹터/KOSPI 갱신
  ├ FG 스코어링                        boost + defense 병렬 (국면 전환 대비 양쪽 상시 생성)
  ├ weighted_rank 후처리               3일 가중 + ✅ 검증
  ├ 확신가중 fusion                    보유 종목 컨센서스 수집 (표시 제안용)
  └ 텔레그램 발송 + state/ git push    발송 원장은 append-only
```

- **재무**: DART 주도(그날 공시 종목만 증분 수집) + FnGuide 보충 + 계정 오매핑 자동 정정.
  성장 팩터는 전부 **실제 공시일 기준 PIT** — 백테스트에 미래 데이터가 새지 않습니다.
- **가격**: KRX 공식 수정주가. (자작 권리락 보정은 진짜 폭락까지 "무상증자겠지"라고 펴버리는
  잘못된 보정으로 판명되어 폐기.)
- **안전망**: 랭킹 종목 수가 150 미만이면 데이터 사고로 간주하고 발송 차단 + 재시도.
  조용한 실패 경로는 전수 감사(SPOF audit)로 전부 개인봇 경보화했습니다.

## 7. 검증 철학

이 저장소의 절반은 전략 코드가 아니라 **검증 흔적**(`backtest/_*.py`, `research/`)입니다.
룰 하나를 바꾸려면 통과해야 하는 관문:

1. **7.4년 풀 백테스트** (2019-01~) — 단기 검증만으로 배포 금지. 6주 검증으로 배포했던
   권리락 보정이 7.4년 격리 BT에서 Calmar를 반토막 내는 걸 발견한 뒤 철칙이 됐습니다.
2. **Leave-One-Winner-Out (LOWO)** — 최대 승자 1~2종목을 빼고도 개선이 남는가?
   소표본 백테스트의 edge는 대부분 슈퍼위너 착시입니다.
3. **워크포워드** — 강세장(19-21)·약세장(22-23)·최근(24-26) 블록별로 train 승자가
   test에서도 이기는가? 특히 약세장 블록이 최우선 관문입니다.
4. **인접 안정성** — 파라미터 ±1칸이 고원인가, 고립 스파이크(과적합)인가?
5. **노이즈 문턱** — 백테스트 Calmar ±0.10은 노이즈. 못 넘으면 채택 안 함.
6. **production 정합** — 검증 하니스가 실제 운영 코드와 같은 시맨틱스인가? 페널티 효과는
   편법 계산 금지, 킬스위치 켜고 전체 재생성으로만 측정.

그리고 문화적 규칙: **재량 개입 금지.** 룰을 따르거나, 검증으로 룰을 바꾸거나 — 둘 사이의
임의 개입("이번만 미리 사자")이 최악입니다. 기각된 아이디어는 근거와 함께
[기각 아카이브](docs/VERSION_HISTORY.md#기각-아카이브-재시도-금지)에 기록해 같은 검증을
두 번 하지 않습니다.
