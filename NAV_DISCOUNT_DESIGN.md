# NAV 디스카운트 별도 트랙 — 설계 (Phase 5 미래 작업)

> 2026-05-16 자율 작업 중 설계 메모. 본격 구현은 사용자 도래 후.

## 배경

산업지주사 (SK스퀘어, LG, CJ, HD현대 등)는:
- 자회사 지분 보유 → 시가총액 의 큰 부분이 자회사 지분 가치
- v80.x의 V/Q/G/M 점수 체계는 손익 기반 → 지주사 NAV 디스카운트 못 잡음
- 예: SK스퀘어 5/16 시가총액 = NAV의 56% (44% 디스카운트)
- 자회사(SK하이닉스) 강세 + SK스퀘어 횡보 → 디스카운트 확대 → 매수 기회

## 측정 지표

```
NAV = Σ (자회사 지분율 × 자회사 시가총액) + 비상장 사업 가치 + 순현금
NAV 디스카운트율 = (시가총액 − NAV) / NAV
```

- < -30%: 강한 디스카운트 (매수 후보)
- < -40%: 매우 강한 디스카운트
- > +10%: 프리미엄 (매수 X)

## 데이터 소스

### 자회사 지분율
- **DART 사업보고서 — 종속회사 명세** (KIS-Code, 지분율)
- OpenDartReader: 미지원 endpoint, raw HTML/XML 파싱 필요
- 또는 FnGuide 종속회사 화면

### 자회사 시가총액
- pykrx (이미 있음) 또는 FnGuide
- 비상장사: 사업가치 추정 필요 (PER 기반 또는 NAV)

### 비상장 사업 가치
- 어려운 부분 — 사업회사의 EV 추정
- 단순화: 비상장사 비중 작은 지주사부터 (NAVER 라인, CJ제일제당 등 대부분 상장)

## 구현 단계

**Phase 5.1 — 표본 (1주)**
1. SK스퀘어 단일 종목 NAV 계산 표본
2. 자회사 지분율 수동 입력 (SK하이닉스 20.07%)
3. 매일 NAV / 디스카운트율 계산 + 시계열 저장
4. 텔레그램 개인봇 NAV 알림 (디스카운트 < -40% 시)

**Phase 5.2 — 화이트리스트 확장 (1주)**
1. 1차 화이트리스트 8~9종목 수동 매핑
2. 동시 모니터링
3. 매수 후보 리스트 별도 메시지

**Phase 5.3 — 자동화 (2주)**
1. DART 사업보고서 — 종속회사 자동 파싱
2. 정기 업데이트 (분기마다 갱신)
3. 비상장 사업가치 PER 기반 자동 추정

## 위험 / 제약

1. **NAV 추정 불확실**: 비상장사 사업가치 PER 추정 노이즈
2. **유동성 함정**: 디스카운트 영원히 안 줄어들 수도 (해소 트리거 필요)
3. **시간 부담**: 구현 약 10시간, 분기마다 종속회사 정보 갱신
4. **모자관계 중복**: 자회사 (예: SK하이닉스) + 지주사 (SK스퀘어) 동시 매수 시 분산 효과 ↓
