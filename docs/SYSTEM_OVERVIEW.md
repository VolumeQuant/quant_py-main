## 1. 국면 게이트

**규칙**: KOSPI 종합지수의 20일 이동평균이 80일 이동평균 위면 공격(boost), 아래면
방어(defense). 신호가 **5거래일 연속** 유지돼야 전환을 확정하고, 전환 시 보유 종목을
전량 청산합니다. 방어 모드는 현금 100% — 신규 매수가 없습니다(진입 슬롯 0).

**왜 이 조합인가**: 단기MA × 장기MA × 확인일수 75조합을 전수 재탐색한 결과입니다.
합산 성과 1위(15/80/5)와의 차이는 +0.015로 노이즈였고, 결정타는 **약세장(2022-23)
워크포워드에서 20/80/5가 최소 Calmar 1.40으로 1위**였다는 점입니다. 합산이 더 높은
후보들은 약세장에서 0.92~0.97로 붕괴 — 강세장 과적합이었습니다.

**왜 KOSPI 종합인가**: "코스피는 삼성·하이닉스가 떠받쳐서 왜곡된다"는 합리적 의심이
있어 대안(코스피200, 코스닥150, ex-메가, 동일가중 지수)을 전부 측정했습니다. 전부
기각 — 동일가중은 in-sample에서만 이기고 OOS에서 역전됐고, 코스닥150은 휩쏘 가짜방어가
최악이었습니다. 장기 검증(2000~)에서 이 게이트는 6개 약세장 전부에서 매수보유 대비
MDD를 −7~32%p 줄였습니다 (2008년 −32%p, 유일한 약점은 2011년 같은 빠른 V자).

**표시**: 전환이 진행 중이면(예: 데드크로스 2일째) Signal 푸터에 카운트다운을 띄웁니다.
5일째 확정 후에야 알면 대비를 못 하기 때문 — 단, 매매룰 자체는 5일 확인을 지킵니다.

## 3. 팩터 모델

공격 모드의 점수식:

```
score = 밸류×0.15 + 성장×0.55 + 모멘텀(12m)×0.30
        + 단기모멘텀(10d) z×0.05 + 저변동성 z×0.06
        + min(과열 z, 0)×0.2
        − 함정 페널티들 (4절)
```

**성장(G)이 왜 0.55로 지배적인가**: V/Q/G/M 가중치 12개 시나리오를 재최적화할 때마다
V15·Q0·G55·M30이 1위입니다. 퀄리티(Q)가 0인 것도 옵티마이저의 일관된 선택입니다.
G 내부는 3개 서브팩터 — 매출 YoY(0.4) + 영업현금흐름 변화(0.4) + 매출총이익 성장(0.2),
전부 DART 공시일 기준 PIT.

**가격이 순위에 어떻게 들어가나**: 분기 재무는 90일간 안 변해서 순위가 "박제"되는 문제가
있었습니다. 두 장치가 이를 풉니다:

- **단기모멘텀 + 저변동성 가산** (각 0.05/0.06) — 매일 종가가 순위에 반영됩니다.
  도입 BT: Calmar 2.43→3.06, MDD −31.3→−29.3%.
- **과열 캡** — 매일 "자기 PER = 시총 ÷ TTM 지배순이익"의 단면 z를 계산해 **비싼 쪽만**
  감점합니다. 주가가 실적보다 빨리 오르면 감점이 커져 추격매수를 피하고, 폭락하면 감점이
  풀립니다. 싼 쪽 보상은 왜 없나 — cheap 보상 변형 5종을 전부 측정했는데 top3의 평균
  수익을 깎았습니다(밸류 틸트가 모멘텀 승자를 희석). "과열만 감점"이 유일하게 무해했고,
  도입 BT는 Calmar 3.14→3.83, 워크포워드 최소값 0.69→1.39.

**밸류가 연간 PER인 이유**: TTM 전환을 3차례 끝장 검증했지만 annual ≈ TTM 동등(IC
0.030 vs 0.028)이었고, 고정 운영 config에서는 annual이 소폭 일관 우위였습니다.
"TTM이 낫다"로 보였던 결과는 best-vs-best 선택 편향으로 판명 — 바꿀 이유가 없습니다.

## 4. 함정 페널티 스택

README [4절](../README.md#4-함정-방어--실계좌-손실에서-역설계한-필터들)의 확장판.
각 필터의 발동 조건과 검증 수치:

### lumpiness — 일회성 매출 (v80.29)

```
min(최근 4분기 매출) / max(최근 4분기 매출) < 0.25  →  성장점수 × 0.3
```

디바이스이엔지(분기매출 53→262→86→392억)를 잡기 위해 지표 6종(CV, 추세R², YoY,
스파이크 등)을 비교했는데, 함정 5종과 승자(제주반도체 0.76, 브이엠 0.42)를 깨끗이
가른 건 min/max뿐이었습니다. 킬스위치 켜고 7.4년 전체 재생성으로 결판: **ON 4.24 >
OFF 3.58, 전 구간 우위, 디바이스 매수권 진입 16일→0일, 제주반도체 보존.**
임계 0.25가 칼날입니다(0.35는 무용) — "가장 약한 분기가 가장 센 분기의 ¼도 안 되는
회사"만 정확히 걸립니다. 우상향 급성장 면제 같은 보완은 넓은 표본에서 효과 0으로 기각.

### accruals — 일회성 이익 (v80.25)

```
B = (순이익TTM − 영업현금흐름TTM)/자산 × 100 > 25   ← 이익이 현금으로 안 들어옴
C = max(최근 4분기 영업이익)/영업이익TTM > 0.7        ← 한 분기 쏠림
B AND C  →  성장점수 × 0.24
```

삼지전자(영업이익 292→3,342)·에스에이엠티가 계기. 왜 하드컷이 아니라 ×0.24 소프트인가 —
이런 종목도 주가는 폭등하는 경우가 있어(에스에이엠티 +393%), 완전 추방하면 7.4년 BT가
−39%p 손해였습니다. "삼성전자 아래로 밀어내되, 그래도 강하게 오르면 일부 캡처"가 최적.

### 최근 자본변동(CA) 페널티 (2026-06-18)

최근 126영업일 내 하락형 CA(무상증자/분할/유상증자, raw 가격 −33% 이상 하락갭으로 감지)
가 있으면 점수 −0.3. 이 코호트는 이후 20일 −6%/1년 −13%로 일관 부진(555건). 원래는
가격 왜곡이 우연히 만들던 "회피 알파"였는데, 정직한 수정주가로 바꾸면서 명시적 페널티로
복원한 것입니다. 병합(상승갭)은 페널티가 오히려 해로워 하락형만 잡습니다.

### 얇은 밸류트랩 (v80.27) / QoQ (v80.12) / 펌프 게이트 (v80.31)

- PER>40 & PBR<1.5 & 거래대금<50억 → 제외 (대덕 유형 — 이익 없이 장부만 싼 트랩)
- 초강세장에서 영업이익 QoQ<+20% → G×0.7 (과열장 실적 미달 감점)
- 이격도20>1.4 & 저성장 → 매수 차단 (실적 없는 가격 펌프)

계절성 페널티는 lumpiness의 부분집합으로 판명(7.4년 반례 0건)되어 **제거**했습니다.
중복 필터는 유지비만 들고 언젠가 조용히 죽었을 때 아무도 모릅니다.

## 5. 순위 체계

- **cr (당일 순위)**: 그날의 순수 실력. 표시(궤적 `r2→r1→r0`)에만 쓰고 판단에는 안 씀.
- **wr (3일 가중 순위)**: `cr₀×0.4 + cr₋₁×0.35 + cr₋₂×0.25`. 진입·이탈·모든 판단의
  유일한 기준. 당일 가중 0.4는 "새 정보 반영"과 "하루 노이즈 억제"의 균형점으로,
  50:30:20에서 하향 조정된 값입니다(v80.13).
- **✅ 검증**: cr 상위 30위를 3일 연속 유지 (⏳ 2일 / 🆕 1일). ✅ 종목만 매수 후보.
- 표시 점수(100점 만점)는 고정 앵커 환산으로, 매매와 무관한 순수 표시입니다. 1등이
  항상 100점으로 박제되던 문제를 고쳐, 약한 날 1등은 50점대·괴물장 1등은 100점이 나옵니다.

## 6. 매매 룰

| 룰 | 값 | 채택 근거 (요약) |
|---|---|---|
| 진입 | ✅ 중 wr ≤ 3 | E2(빡빡)·E4(느슨) 모두 열위 |
| 이탈 | wr > 5 | 아래 상술 |
| 슬롯 | 3, 균등비중 | 4~5슬롯 전체 열위. **3슬롯 분산 자체가 포지션 리스크 컨트롤** — 비중 차등(40/40/20)은 약세장 Calmar 1.42→0.76 붕괴로 기각 |
| 손절 | −15% | 테일 보험 (README 5절). 갭 하락 시 −17~20% 체결 가능성 명시 |
| 트레일링 스탑 | 없음 | 회전 −39%인데 Calmar·MDD 모두 개선돼서 제거 (v80.21) |
| 재진입 쿨다운 | 이탈(순위/SL 공통) 후 10거래일 | 되사기 코호트 승률 29%/−2.05%. K 재스윕: 3~5일은 손실, 10~20일 고원. SL 이탈만 면제하는 변형은 차이 없음(발동 9건뿐)이라 단순 유지 |
| 빈 슬롯 | 승격 금지 (4위로 안 채움) | 승격 허용 시 4.77→3.93 — 약한 후보를 사느니 현금 |
| 국면 전환 청산 | 쿨다운 미적용, 재진입은 한 번에 전량 | 분할매수는 BT 동급(리스크도 동일) — 심리 장치일 뿐 알파 아님 |

**이탈 임계의 역사가 좋은 교훈입니다**: X4(v80.17) → X6(v80.24) → X5(v80.33)로
움직였는데, 변덕이 아니라 **다른 룰과의 상호작용**입니다. 과열 캡이 생기자 "빨리 팔기"가
중복이 돼 X6이 최적이 됐고, 섹터 브레드스가 약세 방어를 넘겨받자 다시 X5로 조여도
안전해졌습니다. 브레드스 OFF 하니스에서는 지금도 X6이 최적으로 재현됩니다 — 즉 X5는
**브레드스 규율을 실제로 지킬 때만** 유효한 선택입니다.

## 7. 섹터 브레드스

**규칙**: 23개 KRX 섹터의 동일가중 지수 중 200일선 위 비율이 35% 미만으로 3거래일
지속되면 "노출 50% 축소"를 권고, 35% 이상 5거래일(보수적 비대칭)이면 복귀를 알립니다.
매매 신호(종목 3개)는 불변 — 사이징 권고이며, 시스템 수익률 리플레이에는 기계 반영됩니다.

**왜 전량이 아니라 50%인가**: 브레드스는 저정밀 신호라(과거 발동의 상당수가 헛방어)
binary 전량 방어는 협소 강세장(메가캡만 오르는 장)에서 −23%p 손해였습니다. "확실한
게이트는 전량, 애매한 브레드스는 절반"이 검증 결과입니다 (도입 BT: Calmar 4.08→4.36,
약세 MDD 24.7→19.2%).

**왜 조기경보가 아닌가**: US 시스템에서는 브레드스가 게이트보다 9개월 일찍 약세를 잡지만,
그건 US 게이트(200일선)가 느리기 때문입니다. KR 게이트(20/80 크로스)는 이미 빨라서
브레드스가 오히려 늦습니다. KR에서 브레드스의 실제 가치는 조기경보가 아니라 **"게이트가
못 잡는 느린 약세의 정밀 보호"**입니다. 더 일찍 당기는 변형은 전부 휩쏘 비용이 커서 기각.

## 8. 데이터 파이프라인

```
평일 16:00 — run_daily.py
  git pull → 거래일 판정 → data_refresher → FG(boost/defense 병렬) → wr 후처리
  → conviction fusion → 텔레그램 발송 → state/ push
```

- **DART 수집**은 "그날 공시한 종목만" 증분(list API) — 전종목 재수집으로 인한 타임아웃
  사고를 막습니다. 분기 마감일 폭주 때 finstate가 비면 document API(XBRL)로 자동 폴백
  하며, 반기 보고서 폴백은 8월 시즌 전에 실물 검증을 마쳤습니다(15/15 계정 일치).
- **FnGuide는 보충** — DART 누락 계정을 자동으로 합치고, 계정 오매핑(과거 SG&A가 매출로
  들어가 SK하이닉스가 랭킹에서 사라진 사고)은 항목별 정합성 검사로 자동 정정합니다.
- **PIT 원칙**: 성장 팩터의 모든 분기 데이터는 실제 공시접수일(rcept_dt)이 기준일보다
  앞설 때만 사용 — 백테스트와 라이브가 같은 정보 시점을 봅니다.
- **무소음 실패 제거**: 거래일 오판(pykrx 빈 응답), 지수 캐시 정체, 브레드스 계산 실패
  같은 조용한 실패 경로를 전수 감사해 전부 개인봇 경보로 명시화했습니다. 원칙은
  "실패해도 되지만, 조용히 실패하면 안 된다".
- **외부 API 예절**: pykrx/DART 병렬 호출 금지, 1초 sleep 순차. (차단당해 본 경험칙.)

## 9. 메시지 레이어

README [8절](../README.md#8-텔레그램-메시지)과 동일하되, 설계 의도 두 가지만 추가:

- **검문소가 존재하는 이유**: top20에 새로 들어온 종목을 매번 수동 검색하는 대신,
  시스템이 쓰는 것과 **동일한 함정 수치**(lumpiness min/max, accruals B/C, 최근 CA,
  과열 감점)로 자동 건강진단을 붙입니다. 푸터의 "필터 생존 모니터"(오늘 상위권에서
  페널티 발동 종목 수)는 필터가 조용히 죽는 사고(6/18 유형)의 카나리아입니다.
- **확신가중이 표시 제안에 머무는 이유**: 기대성장(선행 EPS) 알파는 실측상 진짜지만
  (직교 IC 0.170), 검증이 look-ahead proxy 상한입니다. 실측 컨센서스를 60일+ 누적한
  forward 검증을 통과해야 자동 가중으로 승격합니다 — 그 전까지 종목 선택과 시스템
  수익률은 건드리지 않습니다.

## 10. 메타 자산배분

시스템 코드 밖, 운용자 판단 영역이지만 시스템 설계와 맞물려 있어 기록합니다:

- **현금 20% / 시스템 80%** (시스템 내 KR 60 : US 40). 분기 첫 거래일 정기 + ±5%p 밴드
  리밸런싱. 밴드가 자동 저가매수/이익실현 역할을 하므로 **재량 줍줍은 금지**입니다.
- 현금 버퍼는 수익 장치가 아니라 **리스크 다이얼**입니다. Calmar 최적 현금비율은 기간
  의존(폭등기 0%, 정상+약세기 30~40%)이라 정답이 없고, 성장 목표와 절충해 20%.
- US 40%의 핵심 근거는 분산이 아니라 **달러 쿠션** — KR/US는 위기에 같이 떨어지지만
  (corr 0.7~0.9), 원화 약세가 원화 기준 손실을 줄입니다 (2008: KOSPI −50% vs 나스닥
  원화환산 −28%).
- top3가 전부 반도체여도 시스템에 손대지 않습니다. 시스템은 상대강도 추종이라 섹터가
  꺾이면 자동 로테이션되고, 쏠림 방어는 이 메타 층(3슬롯/브레드스/현금·US)의 일입니다.

## 11. 검증 철학과 사고 이력

검증 관문 6가지는 README [7절](../README.md#7-검증-철학) 참조. 그 관문들이 어떤 사고에서
나왔는지가 이 시스템의 진짜 문서입니다 — [VERSION_HISTORY.md의 "주요 사고와 교훈"

- 편법 하니스 2겹이 부호까지 뒤집음 (lumpiness 초기 검증) → **킬스위치 풀재생성만 인정**
- n=7 소표본이 유망해 보임 → n=3,751에서 효과 0 → **넓은 표본 없이 배포 금지**

타이트 SL 등)는 [기각 아카이브](VERSION_HISTORY.md#기각-아카이브-재시도-금지)에 근거와
