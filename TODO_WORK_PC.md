## 문제 1: DART 2018~2019년 재무 미수집 → Growth=0

Growth TTM YoY = 최근4분기합 / 전년동기4분기합.
2020년 Q1 Growth를 구하려면 2019년 Q1이 필요하고, TTM에 2018년 Q2~Q4도 필요.
현재 DART 재무가 2020-03-31부터만 있어서 2020~2022 Growth가 전부 0.

**작업:**
- `dart_collector.py`의 `fetch_single(ticker, 2018, 2019)` 로 2018~2019 전체 수집
- 1518종목 × 2년 = ~3000건 API 호출 (일일 한도 10,000건 이내)
- 수집 후 기존 fs_dart 캐시에 merge
- 완료 후 bt_2020~2023 재생성 → Growth 정상 계산 확인

## 문제 2: 프로덕션 vs 백테스트 결과 불일치

같은 기간, 같은 진입 날짜인데 프로덕션과 백테스트 시뮬레이터 결과가 크게 다름.

**원인 후보:**

### 2-1. 가격/시가총액 소스 차이
- 프로덕션: pykrx에서 당일 시가총액 실시간 조회 → PER/PBR 계산
- 백테스트: market_cap 캐시 파일 사용 (find_nearest_cache)
- 같은 날이라도 시가총액 수치가 다를 수 있음 → PER/PBR → Value 점수 → 순위 차이

### 2-2. 유니버스 차이
- 프로덕션: pykrx에서 당일 시총/거래대금 실시간 조회
- 백테스트: market_cap 캐시의 find_nearest_cache
- 캐시 날짜가 정확히 일치하지 않으면 유니버스 자체가 달라짐

### 2-3. 섹터 데이터 차이
- 프로덕션: 당일 KRX 섹터 조회
- 백테스트: krx_sector 캐시 (find_nearest_cache)
- 모멘텀은 섹터 내 z-score라서 섹터 매핑이 다르면 M점수가 달라짐

### 2-4. MA120 필터 차이
- 프로덕션: 당일까지 OHLCV로 MA120 계산
- 백테스트: OHLCV 캐시 기간이 짧으면 120일 미만 → 필터 스킵
- 초기 날짜에서 MA120 필터 통과 종목이 달라질 수 있음

### 2-5. WR 동점 처리
- 프로덕션(ranking_manager): WR 동점 시 score 높은 순 tiebreak
- 백테스트 시뮬레이터: 동점 처리가 다를 수 있음
- 경계선(5등 부근) 종목이 동점이면 진입/미진입이 갈림

**해결 방향:**
- 프로덕션 ranking JSON과 백테스트 ranking JSON을 같은 날짜로 비교
- 종목별 점수/순위 diff → 어디서 차이 나는지 특정
- fast_generate_rankings.py를 create_current_portfolio.py와 완전 동일하게 맞추기
